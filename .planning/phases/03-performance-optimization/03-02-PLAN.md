       @Override
       public boolean mightContain(String cacheName, String key) {
           if (cacheName == null || key == null) {
               return false;
           }

           String bloomKey = bloomKey(cacheName);
           String cacheEntryKey = cacheName + "::" + key;

           try {
               int[] positions = hashPositionCache.get(cacheEntryKey,
                   k -> hashStrategy.positionsFor(key, config));

               // ... rest of existing logic using cached positions
           } catch (Exception e) {
               // ...
           }
       }
       ```

       @Override
       public void add(String cacheName, String key) {
           if (cacheName == null || key == null) {
               return;
           }

           String bloomKey = bloomKey(cacheName);
           String cacheEntryKey = cacheName + "::" + key;

           try {
               int[] positions = hashPositionCache.get(cacheEntryKey,
                   k -> hashStrategy.positionsFor(key, config));

               // ... existing Redis pipeline code
           } catch (Exception e) {
               // ...
           }
       }
       ```

    Key insight: Both mightContain() and add() use the SAME cached positions. Using cache.get() with a loader function ensures:
       - If key exists in cache, returns cached value (no recomputation)
       - If key not in cache, computes via loader and caches automatically
       - Thread-safe via Caffeine's internal synchronization
